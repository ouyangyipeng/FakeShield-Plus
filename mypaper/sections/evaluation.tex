\section{Evaluation}
\label{sec:evaluation}

\subsection{Experimental Setup}

\subsubsection{Hardware and Software}

All experiments are conducted on a server with 4$\times$NVIDIA A100-PCIE-40GB GPUs, Intel i9-13900H CPU, and 32GB RAM (WSL2). The software environment includes PyTorch 1.13.0+cu117, transformers 4.28.0, mmcv-full 1.4.7, bitsandbytes 0.41.3.post2, and triton 2.1.0.

\subsubsection{Datasets}

\begin{itemize}
    \item \textbf{CASIA1+}~\cite{casia1}: 4 test images with ground truth masks for pipeline and localization evaluation. Includes splicing and copy-move forgeries.
    \item \textbf{SD\_inpaint}~\cite{fakeshield2025}: 17 authentic images from the evaluation set for cross-domain generalization testing. Downloaded from HuggingFace Hub.
\end{itemize}

\subsubsection{Evaluation Metrics}

\begin{itemize}
    \item \textbf{End-to-End Success Rate}: Percentage of images successfully processed through the entire DTE-FDM $\rightarrow$ MFLM pipeline
    \item \textbf{VRAM Usage}: GPU memory consumption in GB
    \item \textbf{Inference Latency}: Average time per image in seconds
    \item \textbf{Detection Accuracy}: Correct classification of fake vs. real images
    \item \textbf{IoU}: Intersection over Union for localization accuracy
    \item \textbf{F1 Score}: Harmonic mean of precision and recall for localization
\end{itemize}

\subsubsection{VRAM Analysis}

INT8 quantization reduces per-GPU VRAM from 6.94 GB to 3.98 GB (43\% reduction). However, the total system VRAM is 13.36 GB due to bitsandbytes' internal dequantization buffers. This trade-off is beneficial for multi-GPU deployments where per-GPU memory is the bottleneck.

INT4 quantization fails with Out-Of-Memory errors on our 40GB A100 GPU. The loading process requires intermediate FP32 buffers that exceed available memory, suggesting that INT4 for 13B models requires either larger GPUs (80GB) or CPU offloading.

\subsubsection{Latency Analysis}

INT8 inference is 48\% slower than FP16 (52.26s vs 35.18s per image). This overhead stems from dequantization operations during forward passes. The latency-memory trade-off should be chosen based on deployment constraints. As shown in Figure~\ref{fig:latency_comparison}, the latency increase is consistent across all test images.

\subsubsection{Accuracy Preservation}

Both FP16 and INT8 produce identical detection results (3 FAKE, 1 REAL), confirming that INT8 quantization preserves detection accuracy. This aligns with prior findings on INT8 quantization for LLMs~\cite{llm_int8}.

\subsection{Cross-domain Generalization}

We evaluate the extended DTG (6-class) on the SD\_inpaint dataset. The original DTG (3-class) classifies all 17 authentic images as ``AIGC'', while the extended DTG provides finer-grained classification:

\begin{itemize}
    \item 15 images classified as ``SD\_inpaint'' (correct identification of SD-generated content)
    \item 2 images classified as ``Photoshop''
\end{itemize}

The extended DTG achieves 75\% accuracy on the small training subset (13 training, 4 test samples), demonstrating the feasibility of extending the domain tag taxonomy. With more training data, we expect higher accuracy for the new classes.

The IoU evaluation framework enables systematic comparison of localization methods and provides quantitative metrics for future improvements.
